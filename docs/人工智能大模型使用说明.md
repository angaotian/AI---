## （1）AI 使用说明

### 1.1 使用的模型与工具（名称 / 版本 / 日期）

| 类别 | 名称 | 版本或配置说明 | 引用日期（文档撰写） |
|------|------|----------------|----------------------|
| 云端大模型 API（可选） | 秘塔 AI（MetaSo）兼容 OpenAI Chat Completions | 接口路径形如 `/v1/chat/completions`；模型名由环境变量 `MITA_AI_MODEL` 指定（示例：`metaso-chat`，以实际控制台为准） | 2026-05-05 |
| 本地深度学习推理 | TensorFlow.js | `@tensorflow/tfjs-core` 等 `^4.22.0` | 2026-05-05 |
| 本地人脸与表情分析 | face-api.js | `^0.22.2`（依赖上述 TFJS，在浏览器 WebGL 后端运行） | 2026-05-05 |
| 三维可视化（非生成式大模型） | three.js | `^0.183.2`（人体模型展示，不计入「大模型」但属于智能可视化相关） | 2026-05-05 |
| 编程与文档辅助（开发过程） | DeepSeek（网页/App/API 等对话式助手） | **具体模型名称与版本以所用软件「关于」或会话界面为准** | 2026-05-05 |

**补充**：未配置 `MITA_AI_API_URL` / `MITA_AI_API_KEY` 时，「AI 医生」问答由服务端**规则引擎**与关键词命中逻辑应答，不调用外部大模型。

### 1.2 在哪些环节应用了 AI

| 环节 | 说明 |
|------|------|
| 构思 | 借助 DeepSeek 等 AI 工具辅助梳理模块划分（路由、布局、前后端分工）与接口清单。 |
| 生成 | （运行时）可选调用秘塔 AI 生成问诊回复；（开发时）用 AI 辅助生成部分 Vue/Express 样板代码与错误处理草稿。 |
| 润色 | 对界面文案、注释、`docs` 下说明文档进行用语统一与缩写优化（最终以人工定稿为准）。 |
| 编程 | 在 DeepSeek 等环境中通过对话描述需求，辅助编写与调试 `server/index.js` 路由、Vue 页面与联调。 |
| 数据分析 | 本项目健康大盘、风险页等为**演示数据**或前端计算，**未**使用大模型做临床级数据分析；知识图谱为前端内置演示推理。 |
| 图像 / 音频生成 | **未使用**文生图、文生音频类生成模型；多模态页使用摄像头实时帧 + 本地 face-api 模型，语音侧以浏览器能力为主，无额外云端语音大模型依赖。 |

### 1.3 学生把关工作与结果验证

| 把关项 | 做法 |
|--------|------|
| 功能正确性 | 按 `docs/模块测试报告.md` 中各模块用例（含 **AI-01～AI-03**）做手工回归；前后端联调验证 JWT、空输入、错误码。 |
| 医疗与安全边界 | 界面与文案强调**辅助参考、不替代诊疗**；云端回复仅作补充，且服务端 system 提示约束为「严谨的医疗健康助手」；遇急诊相关关键词时规则引擎侧提高风险等级（如胸闷胸痛场景）。 |
| 外部 AI 可用性 | 未配置 API 时验证走规则引擎路径；配置后抽查返回 JSON 字段 `source`、`mita.used` 是否与预期一致。 |
| 本地模型 | 多模态页验证 face-api 模型加载成功/失败提示；摄像头权限被拒绝时的体验。 |
| 隐私与密钥 | API Key **仅放本地 `.env`**，勿提交到版本库；正式报告中可对密钥打码。 |

**正确性说明**：大模型输出可能存在幻觉或过时信息，本项目通过「规则引擎兜底 + 医疗免责声明 + 人工测试」三重约束；**不保证**医学结论准确性，仅作课程作品演示。

## （2）关键提示词与输出样例

下列共 **8 条**：前 **5 条**为开发阶段与 DeepSeek 协作的**代表性提示**（请替换为你自己的真实对话摘要）；后 **3 条**为系统运行时已落地代码中的**真实提示与响应结构样例**。

### 2.1 开发阶段（代表性提示 — 请按实际修改）

**对话 1 — 架构与接口**

- **提示（示例）**：「项目用 Vue3 + Vite 前端、Express + MySQL 后端，需要 JWT 登录和 REST API，请列出用户注册登录、健康指标、AI 问诊 POST 路由的建议路径与请求体字段。」
- **输出要点**：得到路由命名与字段草稿后，由学生对照最终实现逐个删减或改名。

**对话 2 — AI 问诊后端**

- **提示（示例）**：「在 Express 里增加 `/api/doctor/consult`，先根据问题关键词返回 riskLevel 与本地 answer，若存在环境变量则使用 fetch 调用 OpenAI 兼容的 chat completions，并解析 choices[0].message.content。」
- **输出要点**：学生合并入现有 `authenticateToken`、错误处理与秘塔多路径兼容逻辑（见 `server/index.js`）。

**对话 3 — 多模态页面**

- **提示（示例）**：「用 face-api.js 在 Vue3 里加载 tinyFaceDetector 与 faceLandmark，在 video 上画框并打印表情概率。」
- **输出要点**：学生调整模型路径、加载失败提示与页面样式以匹配 `MultiModal.vue`。

**对话 4 — 文档与测试表**

- **提示（示例）**：「根据现有模块列表生成 Markdown 模块测试表框架，列：用例编号、步骤、预期。」
- **输出要点**：学生按实际页面修正编号与预期（见 `docs/模块测试报告.md`）。

**对话 5 — 排错**

- **提示（示例）**：「前端请求 localhost:3002 出现 CORS 或 401，Express 应如何配置 cors 与鉴权中间件顺序？」
- **输出要点**：学生结合实际 `vite` 代理与 `Authorization` 头验证。

### 2.2 产品运行时（摘自本项目代码，无需改写）

**对话 6 — System 提示（固定）**

- **提示（role: system）**：`你是严谨的医疗健康助手，请给出清晰且安全的建议。`  
- **出处**：`server/index.js` 中调用 Chat Completions 时的 `messages[0].content`。

**对话 7 — User 提示（随用户输入变化）**

- **提示（role: user）**：`最近一周血压偏高，偶尔头晕，需要关注哪些问题？`（示例问题，可替换任意非空字符串）  
- **说明**：与界面输入框一致，经 `trim()` 后传入。

**对话 8 — 输出样例（JSON 结构节选）**

以下为接口返回形态示例（大模型未启用时 `mita.used` 为 `false`，`source` 为规则引擎）：

```json
{
  "answer": "建议先监测血压并记录波动；若伴随胸痛、言语不清或肢体无力，请立即就医。",
  "riskLevel": "中",
  "source": "mita-ai+knowledge-graph",
  "knowledgeHits": ["头晕", "血压"],
  "mita": {
    "used": true,
    "answer": "（此处为秘塔返回的正文摘要，以实际 API 为准）"
  },
  "timestamp": "2026-05-05T12:00:00.000Z"
}
```
