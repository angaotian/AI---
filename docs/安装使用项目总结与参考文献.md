# 第五章　安装及使用

## 5.1　环境要求

| 类别 | 要求说明 |
|------|----------|
| 操作系统 | Windows 10/11、macOS 或 Linux（64 位） |
| 运行时 | **Node.js** 建议 LTS 版本（如 18.x / 20.x），含 npm |
| 数据库 | **MySQL** 5.7+ 或 8.0+，服务已启动并可本机连接 |
| 浏览器 | Chrome、Edge 等 Chromium 内核浏览器（多模态采集需摄像头/麦克风权限） |
| 可选 | 配置服务端 `.env` 中的 `MITA_AI_API_URL`、`MITA_AI_API_KEY` 等，以启用外部 AI 问答 |

## 5.2　默认安装步骤

1. **获取代码**  
   将项目解压或克隆至本地目录（路径中尽量避免仅含特殊字符，以免终端编码问题）。

2. **安装前端依赖**  
   在项目根目录执行：
   ```bash
   npm install
   ```

3. **安装后端依赖**  
   ```bash
   cd server
   npm install
   cd ..
   ```

4. **配置数据库**  
   - 在 MySQL 中创建数据库：`health_management`（名称需与配置一致）。  
   - 编辑 `server/database/config.js`，填写本机的 `host`、`port`、`user`、`password`、`database`。  
   - 首次启动后端时，程序会按 `server/database/schema.sql` 检查并创建缺失表结构；若仓库提供初始化脚本，可按说明导入示例数据。

5. **（可选）环境变量**  
   在 `server` 目录下创建 `.env`，按需配置 AI 接口等变量；不配则使用内置规则引擎应答。

## 5.3　启动与默认访问

1. **启动后端**（需先保证 MySQL 可用）  
   在项目根目录：
   ```bash
   npm run server
   ```
   默认监听 **3002** 端口（以 `server/index.js` 为准）。

2. **启动前端开发服务**  
   另开终端，在项目根目录：
   ```bash
   npm run dev
   ```
   根据 `vite.config.js`，开发服务器默认 **3001** 端口。

3. **访问**  
   浏览器打开 `http://localhost:3001`（或控制台输出的实际地址）。前端通过 `VITE_API_BASE_URL` 或默认 `http://当前主机名:3002/api` 访问后端。

## 5.4　生产构建（可选）

```bash
npm run build
```

生成静态资源至 `dist/`，可由任意静态服务器或反向代理托管；需将 API 指向已部署的后端地址（环境变量 `VITE_API_BASE_URL`）。

## 5.5　典型使用流程

1. **注册 / 登录**  
   打开应用后若未登录，跳转至登录页；注册新用户或使用已有账号登录，前端保存 JWT 至本地存储。

2. **浏览健康数据**  
   在侧边栏进入「健康大盘」「LQI 详情」「健康报告」等，查看指标、趋势与报告型内容。

3. **三维孪生与分析**  
   进入「3D 数字孪生」查看人体模型与器官标注；进入「知识图谱推理」「多模态采集」进行图谱查询或多模态采集（需授权摄像头等）。

4. **家庭与系统**  
   在「家庭守护」中维护家庭成员；在「消息中心」「系统设置」中查看消息与调整偏好。

5. **退出**  
   使用侧栏退出登录，清除本地令牌并返回登录页。

# 第六章　项目总结

本作品为个人健康管理系统的前后端一体化原型，开发过程中将界面划分为「个人健康—智能分析—家庭与消息—系统运维」等模块，配合 Vue Router 与布局组件实现清晰的信息架构。技术选型上采用 Vue 3 组合式 API 与 Vite 提升迭代效率，Express 与 MySQL 承担认证、健康指标、家庭成员与多模态记录等持久化，前后端通过 JWT 与 REST 接口协作，便于后续替换为云部署或微服务。

协调与任务分解方面，宜先固定路由与数据契约，再并行开发页面与接口，可减少联调返工。实际困难主要包括：浏览器对摄像头与本地 AI 模型的权限与体积限制、Three.js 场景与业务数据的绑定与性能平衡，以及知识图谱前端可视化与推理逻辑的耦合度控制。通过分阶段交付（先通登录与列表，再叠加图表与 3D、最后接多模态与 AI），整体风险可控。

水平与认识上，对前端状态管理、Canvas/Three 渲染管线以及服务端鉴权有了更系统的实践；也认识到医疗类应用必须在文案与交互上强调「辅助决策、不替代诊疗」的边界。后续演进可考虑：完善健康报告的导出与打印、对接可穿戴设备或医院开放平台、在服务端统一审计日志与限流、将 AI 问答与临床指南版本绑定并做合规评估；若面向推广，还需产品化运营、隐私合规与用户协议等配套，而非仅功能堆叠。

（本章篇幅控制在约一页 A4 纸以内，可按学校格式微调字号与行距。）
