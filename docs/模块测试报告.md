# 个人健康管理系统 — 模块功能测试报告

| 项目 | 说明 |
|------|------|
| 系统名称 | 个人健康管理系统（router_demo） |
| 前端 | Vue 3 + Vue Router + Vite（开发默认端口 3001） |
| 后端 | Node.js + Express（默认端口 3002），JWT 认证 |
| 可选能力 | 配置 `MITA_AI_API_URL` / `MITA_AI_API_KEY` 后 AI 问答可走外部大模型 |
| 结果说明 | **符合预期**：与预期一致；**基本符合预期**：核心可用，含原型数据、演示模式或未接全能力 |
| 实际结果填写 | 下表以「符合预期」为模板；**RPT-02、MEM-04** 等条目的预期已注明能力边界，实测可改为「基本符合预期」并备注 |

## 4.1.1 用户认证模块测试

**范围**：`/login`（`Login.vue`），REST：`POST /api/auth/register`、`POST /api/auth/login`；`localStorage` 保存 `token`；路由守卫未登录跳转登录页。

| 测试 ID | 测试用例 | 预期结果 | 实际结果 |
|---------|----------|----------|----------|
| AUTH-01 | 有效凭据登录 | 登录成功，返回 JWT，前端存储 token，跳转至主应用（默认进入健康报告等已登录页） | 符合预期 |
| AUTH-02 | 错误密码登录 | 提示「用户名或密码错误」或等价错误信息，不写入 token | 符合预期 |
| AUTH-03 | 用户注册 | 填写用户名、邮箱、密码等必填项后注册成功，可随后登录 | 符合预期 |
| AUTH-04 | 未登录访问受保护路由 | 无 token 时访问业务页被重定向至 `/login` | 符合预期 |
| AUTH-05 | 已登录访问登录页 | 持有有效 token 访问 `/login` 时跳转离开登录页（如进入 `/health-report`） | 符合预期 |

## 测试结论

1. **用户认证、健康业务页、数字孪生、知识图谱、多模态、家庭、消息、设置**等模块应按上表逐条执行测试，「实际结果」列填写 **符合预期** 或 **基本符合预期** 并必要时备注环境（浏览器权限、MySQL、模型文件等）。  
2. **多模态采集**强依赖本机摄像头/麦克风与 face-api 模型加载；**知识图谱**推理与图谱数据为前端内置知识演示，与真实临床知识库版本需区分。  
3. **AI 问诊**为服务端规则 + 可选外部 API；**健康报告导出**若仍为占位，需在实现下载/打印后回归 **RPT-02**。  
4. **后台运维**页面为运维可视化原型，OPS-02/OPS-03 达标含义为「交互与展示正常」，不等同于生产级监控。
